# SceneScript Animation Engine Documentation

A Remotion + WebGL pipeline for producing YouTube Shorts–style animated videos from a single `animation.yml` file. Images are sourced from a texture atlas; the engine composites, animates, and renders them frame-by-frame at 60 fps.

## Quick Start

```bash
npm start          # starts the hot-reload preview server
# edit animation.yml → changes appear instantly in the browser
```

For final export, use Remotion's render (via UI or script)

## Resolution & Composition (`src/Root.tsx`)

Resolutions are defined in the `RES` map:

| Key            | Width  | Height |
|----------------|--------|--------|
| `r1080p`       | 1920   | 1080   |
| `r4k`          | 3840   | 2160   |
| `r720p`        | 1280   | 720    |
| `shorts`       | 1080   | 1920   |
| `shorts_alt`   | 720    | 1280   |
| `shorts_split` | 1140   | 1140   |

Switch resolutions by changing the `_res` assignment:

```ts
const _res = RES.shorts          // ← change this line
```

The engine enforces even pixel dimensions automatically (`Math.floor(x / 2) * 2`).

`FPS` is set to **60**. The default composition length is **180 seconds** (`FPS * 180`).

## Texture Atlas

All sprites must exist in a single texture atlas, generated by the author's own tool and placed at:

```
public/texatlas/texatlas.png      ← sprite sheet image
public/texatlas/atlasmeta.cson    ← frame metadata (CSON format)
```

Reference source:
- https://github.com/cosmicsplendor/remotion-3d/blob/master/public/texatlas/atlasmeta.cson
- https://github.com/cosmicsplendor/remotion-3d/blob/master/public/texatlas/texatlas.png

Frames are referenced by their string names (e.g. `baldback`, `fire_0`, `prisonbar`) directly in `animation.yml`. Animated clips reference frame sequences built from a prefix + frame numbers.

## Captions (`src/components/Captioned/captions.yaml`)

To enable or disable captions, use the **Captioned** composition in `Root.tsx`. Captions are defined in `captions.yaml`:

```yaml
- text: a
  duration: 3        # shown for 3 frames

- text: man
  duration: 13

- text: died
  start: 20          # explicit frame start; if omitted, begins right after previous caption ends
  duration: 19
```

- `start` is optional. If not specified, the caption is appended immediately after the previous one ends.
- `duration` is always in **frames** (at 60 fps, 60 frames = 1 second).
- The video source for the Captioned composition is set via `defaultProps.videoSource`.

## `animation.yml` — Top-Level Structure

```yaml
XOffset: 100          # global X translation applied to the entire scene
ZOffset: 500          # global Z depth offset
ORIGIN_Y: 0.6         # default vertical anchor for the camera (0 = top, 1 = bottom)
StartSequence: SETUP  # which EventID to play first
HidePrevious: true    # hide objects from previous events when a new one starts
NoGen: false          # if true, skips all Generator blocks
FOV: 120              # default field of view

Clips: ...            # sprite animation sequences
Modifiers: ...        # reusable procedural motion behaviors
Sequence: ...         # ordered list of cinematic events
```

## Clips

Clips define frame-by-frame sprite animations. They are triggered on an object via `Clip: ClipName` in a keyframe.

```yaml
Clips:
  Fire:
    FrameDuration: 0.025     # seconds per frame
    Prefix: fire_            # prepended to each frame number to form the atlas key
    Frames: [0,1,2,...,29]   # frame indices; can be non-sequential or repeated
    Playback: Loop           # Loop | Once
```

| Property        | Description |
|-----------------|-------------|
| `FrameDuration` | Duration of each frame in seconds |
| `Prefix`        | Atlas key prefix (e.g. `fire_` + `3` → `fire_3`) |
| `Frames`        | Array of frame indices (or string suffixes like `n`, `n2` for the Lip clip) |
| `Playback`      | `Loop` repeats indefinitely; `Once` plays through and stops on the last frame |

**Example — trigger a clip on an object:**
```yaml
Keyframes:
  - Time: 0.35
    Clip: ImpactHot
```

## Modifiers

Modifiers are reusable, procedural animation behaviors (oscillators or value sequences) that can be toggled on/off on any object at any keyframe.

### Oscillator type

Continuously oscillates a property using a sine or cosine wave.

```yaml
Gentle_Breathe:
  Type: Oscillator
  TargetProperty: scale       # property to drive (scale, alpha, position.x, rotation…)
  Waveform: Sine              # Sine | Cosine
  Amplitude: 0.03             # peak deviation from base value
  Frequency: 0.5              # cycles per second
```

### Sequence type

Drives a property through a looping list of values, interpolated over a fixed cycle duration.

```yaml
Walk_Bob:
  Type: Sequence
  TargetProperty: position.y
  Playback: Loop
  Interpolation: sineInOut    # easing between values
  Values: [0, 15, 25, 15, 0, -10, -15, -10]
  CycleDuration: 0.15         # seconds for one full pass through Values
  Amplitude: 1.0              # master scaler multiplied onto all values
```

| Property        | Description |
|-----------------|-------------|
| `TargetProperty`| Dot-notation path: `scale`, `rotation`, `alpha`, `position.x`, `position.y` |
| `Interpolation` | Easing function between sequence steps |
| `Values`        | Normalized shape of the motion; multiplied by `Amplitude` at runtime |
| `CycleDuration` | Seconds for one complete cycle |
| `Amplitude`     | Scales the entire Values array up or down |

**Toggling modifiers in keyframes:**
```yaml
Keyframes:
  - Time: 0
    Modifiers: { Walk_Bob: on, Walk_Rotation: on }
  - Time: 0.75
    Modifiers: { Walk_Bob: off, Walk_Rotation: off }
```

## Sequence (Events)

The `Sequence` array is an ordered list of **events** — each is a self-contained cinematic shot.

```yaml
Sequence:
  - EventID: SETUP
    Duration: 285            # duration in frames (at 60 fps)
    Normalize: true          # normalize keyframe Times to [0..1] fractions of Duration
    ORIGIN_Y: 0.65           # overrides the global ORIGIN_Y for this event
    Camera: ...
    Generators: ...
    Objects: ...
```

| Property    | Description |
|-------------|-------------|
| `EventID`   | Unique string identifier |
| `Duration`  | Length of the event in frames |
| `Normalize` | When `true`, keyframe `Time` values are fractions 0–1 (and can exceed 1 for overshoot). When `false`, `Time` is in seconds |
| `ORIGIN_Y`  | Vertical screen anchor for this event's camera (overrides global) |

## Camera

Each event has a `Camera` block with `Keyframes` keyed by normalized time (or seconds).

```yaml
Camera:
  Keyframes:
    0:
      x: 19.25
      y: 200
      z: 5600
      fov: 140
      Easing: { Position: circInOut }
    0.7:
      x: 19.25
      y: 350
      z: 5675
      fov: 140
```

- `x`, `y`, `z` are world-space camera position.
- `fov` is field of view in degrees.
- `Easing` can target `Position` and `fov` independently per keyframe.

## Objects

Objects are individual sprites placed and animated within an event.

```yaml
Objects:
  - ID: baldback
    Initial:
      pos: { x: 34.2, y: 0, z: 2680 }
      scale: 8
      frame: baldback       # atlas frame name
      scaleX: 0.9           # independent X scale (for squash/stretch)
      alpha: 1.0
      anchor: { x: 0.5, y: 1.0 }   # pivot point (0–1 normalized)
      flip: true            # mirror horizontally
      centerY: true         # anchor to vertical center
      invisibleTill: 0.3    # hidden until this time
      hideAfter: 0.9        # hidden after this time
      fadeinDuration: 0.05  # fade-in duration in normalized time
      fadeoutDuration: 0.1  # fade-out duration in normalized time
      startAlpha: 1.5       # initial alpha (can exceed 1 for brightness boost)
      hidebeforehand: true  # equivalent to invisibleTill: 0
      fogFac: 0             # depth fog influence (0 = no fog)
      Clip: CrackerSpark    # start playing this clip immediately
    Keyframes:
      - Time: 0.55
        Frame: baldfront
        Scale: 6
        ScaleX: 1.1
        Position: { x: 34.2, y: 0, z: 2680 }
        Rotation: 0
        Alpha: 1
        Flip: false
        Easing: { Scale: spring, Position: criticallyDamped, Rotation: quadIn }
        Modifiers: { Subtle_Breathe: on }
        Clip: RedExplosion
```

### Initial Properties

| Property          | Description |
|-------------------|-------------|
| `pos`             | World-space position `{x, y, z}` |
| `scale`           | Uniform scale |
| `scaleX`          | Horizontal-only scale multiplier |
| `scaleY`          | Vertical-only scale multiplier |
| `frame`           | Starting atlas frame name |
| `alpha`           | Opacity (0–1; values >1 add brightness in the WebGL shader) |
| `anchor`          | Pivot `{x, y}` in 0–1 normalized sprite coords |
| `flip`            | Horizontal mirror |
| `centerY`         | Pivot to vertical center |
| `invisibleTill`   | Hidden until this normalized time |
| `hideAfter`       | Hidden after this normalized time |
| `fadeinDuration`  | Fade-in window length |
| `fadeoutDuration` | Fade-out window length |
| `startAlpha`      | Override alpha at the very first frame |
| `fogFac`          | Depth fog contribution |
| `Clip`            | Immediately start this clip |

### Keyframe Properties

All `Initial` properties can be animated in keyframes, plus:

| Property   | Description |
|------------|-------------|
| `Time`     | Normalized (0–1) or seconds depending on `Normalize` |
| `Easing`   | Per-property easing: `{ Scale: spring, Position: cubicOut, Rotation: sineIn, Alpha: step, fov: criticallyDamped }` |
| `Modifiers`| Toggle modifiers: `{ Walk_Bob: on, Wind_Sway: off }` |
| `Clip`     | Switch to a new sprite clip at this keyframe |

## Generators

Generators procedurally spawn multiple objects using a seed and index variable `i`.

```yaml
Generators:
  - IDPrefix: weeds       # generated IDs become weeds_0, weeds_1, …
    Seed: 116             # RNG seed for reproducible randomness
    Count: 50             # number of objects to spawn
    Variables:
      x: "rndRange(97, 98.75)"
      z: 18000 + i * 10
      frame: "rndRange(0, 2) < 1 ? 'weed1': 'weed2'"
      flip: "rndRange(0, 2) < 1 ? true: false"
    Initial: { pos: { x: vars.x, y: 0, z: vars.z }, scale: 5, frame: vars.frame, flip: vars.flip }
```

| Feature         | Description |
|-----------------|-------------|
| `IDPrefix`      | Prefix for generated object IDs (`IDPrefix_0`, `IDPrefix_1`, …) |
| `Seed`          | RNG seed for `rndRange()` calls |
| `Count`         | Total objects to generate |
| `Variables`     | JS expressions evaluated per-object; `i` is the current index, `rndRange(min, max)` is available |
| `Initial`       | Standard object Initial block; reference computed vars via `vars.name` |

Generators support the same `Initial` properties as regular Objects but currently do not support per-object `Keyframes`.

## Easing Functions Reference

| Name               | Character |
|--------------------|-----------|
| `linear`           | Constant speed |
| `step`             | Instant jump (no interpolation) |
| `sineIn/Out/InOut` | Smooth sine curve |
| `cubicIn/Out/InOut`| Cubic acceleration |
| `quartIn/Out`      | Quartic acceleration |
| `quintIn/Out`      | Quintic acceleration |
| `expoIn/Out/InOut` | Exponential |
| `circIn/Out/InOut` | Circular arc |
| `backOut`          | Overshoot then settle |
| `bounceOut`        | Bounce at the end |
| `spring`           | Physics spring (overshoot + oscillate) |
| `criticallyDamped` | Spring with no overshoot |
| `smoothStep`       | Hermite S-curve |
| `smootherStep`     | Smoother S-curve (Ken Perlin) |
| `staircase`        | Stepped discrete jumps |
| `sin`              | Raw sine (alias) |

## Tips & Common Patterns

**Looping walk cycle** — pair a `_Bob` (Y position) modifier with a `_Rotation` modifier, both with matching `CycleDuration`:
```yaml
Modifiers: { Walk_Bob: on, Walk_Rotation: on }
```

**Fire/smoke combo** — layer a looping `Fire` clip behind a `ThickSmoke` clip with a slight `invisibleTill` offset for a realistic ignition.

**Squash & stretch** — animate `ScaleX` and `Scale` with `spring` easing in opposite directions at the same keyframe.

**Z-ordering** — objects are depth-sorted by their `z` coordinate. Place foreground elements at lower Z values than background elements within the same event region.

**Matching event Z ranges** — each event lives in its own Z corridor (e.g. SETUP at z≈2000–2700, Fighting at z≈5600–5900). Keep all objects for an event within its corridor to avoid cross-event bleed.

**`hidebeforehand: true`** is equivalent to `invisibleTill: 0` and is useful for objects that should only appear after a keyframe transition triggers them.
